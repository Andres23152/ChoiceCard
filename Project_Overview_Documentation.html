Project Overview

The HSHP Course Selection System is a dynamic workflow that integrates Microsoft Forms, Power Automate, SharePoint, and GitHub. Its primary function is to:

  • Enable students to select courses in a user-friendly form (branching by grade level).
  • Generate individualized PDFs with each student’s choices.
  • Send automated emails to both students and parents, enabling them to confirm or request changes.
  • Maintain all data in a centralized SharePoint list for record-keeping and reporting.
  • Provide bilingual (English/Spanish) support throughout all communications.

This ensures transparency, reduces administrative overhead, and captures valuable data for each student’s upcoming schedule.

Microsoft Form Structure

A single Microsoft Form with ~50 questions (including branching) captures student data. The first section asks for:

  • ID or “NTD” if new to district
  • Student First/Last Name & Email
  • Parent Email
  • Incoming Grade Level (8th–12th)

Based on grade selection, branching logic presents the relevant subjects and extracurriculars. Here’s an overview:

8th Grade

  • PAP Reading, PAP Social Studies, various math and science options, Health Science, Foreign Language, limited electives.
  • No off-campus extracurricular is offered at this level.

9th Grade

  • English, Math (Algebra 1, Geometry, Algebra 2), Science (Biology, Chemistry), Health Science, Foreign Language, Fine Arts, PE, multiple electives.
  • Off-campus extracurricular question + details if “yes.”

10th Grade

  • English, Math, Science, Social Studies, Health Science, Foreign Language, Fine Arts, PE, Electives, LVN Program interest for junior year, off-campus extracurricular.

11th Grade

  • English, Math, Science, Social Studies, Practicum, Electives, off-campus extracurricular.

12th Grade

  • English, Social Studies, Practicum, Electives, off-campus extracurricular.

The final section captures additional questions or comments for the counselor. Once the student submits, a Power Automate flow is triggered, branching further based on grade level.

System Flows & Logic

Power Automate underpins the entire system, connecting, Microsoft OneDrive, Microsoft Forms with SharePoint, Outlook, and GitHub-hosted pages. When a student submits the form, the Grade Approval Creation Flow runs:

  1. Captures response details from the form.
  2. Uses a Switch Case for each grade (8th–12th).
  3. Determines if the student is new or existing, then updates/creates a SharePoint entry.
  4. Generates an HTML template (grade-specific), converts to PDF, stores in OneDrive, and sends the student an email with a link to confirm choices.

The student’s confirmation link triggers another flow, Student Confirmation Flow, which:

  • Updates the SharePoint item from “not responded” to “confirmed.”
  • Creates a PDF for the newly confirmed status.
  • Sends a parent-facing email with bilingual options to confirm or request changes.

If the parent confirms, the Parent Confirmation Flow marks the item “Parent Confirmed.” If they request changes, the Parent Rejection Flow logs an “Inquiry” status, prompting a counselor follow-up.

A separate GET Request Flow returns student/parent data to static GitHub pages, ensuring real-time dynamic content.

Detailed Flow Explanations

Grade Approval Creation Flow

  • Trigger: Microsoft Form submission.
  • Logic: Switch case on the grade, nested if statements for new vs. existing, create or update SharePoint record, generate HTML -> PDF, store in OneDrive, send email to student.

Student Confirmation Flow

  • Trigger: HTTP POST from student’s “Confirm” link.
  • Actions: Mark SharePoint “Confirmed,” create a second PDF, email parent with 4 possible buttons (View course, confirm, request changes, view course bulletin).

Parent Confirmation Flow

  • Trigger: Parent clicks “Confirm” in their email, hitting a different HTTP endpoint.
  • Actions: Updates SharePoint to “Parent Confirmed,” generates final PDF, sends email to both parent and student, optional counselor CC.

Parent Rejection/Inquiry Flow

  • Trigger: Parent clicks “Request an Inquiry” or “Reject,” calling a separate endpoint.
  • Actions: Logs “Parent Rejected” or “Inquiry” in SharePoint, captures phone number and comments, notifies counselor by email.

GET Request Flow

  • Trigger: GitHub page calls a URL with ?studentID=xxx&FirstName=xxx&LastName=xxx or other params.
  • Actions: Fetch matching record from SharePoint, return JSON with course selections, status, etc. so the static page can populate data in real-time.

Parent Involvement & Multi-Language Support

One of the most significant features is the involvement of parents at a crucial stage of the process. Immediately after the student confirms their choices, a custom email is generated for the parent with bilingual prompts, ensuring that families who speak Spanish can fully participate.

Each parent email includes:

  • Summary of their child’s course selections.
  • Links to confirm or reject/inquire, along with bilingual instructions.
  • Buttons to view a dynamically generated PDF or to read a course bulletin (giving an overview of the classes offered).

On the hosted GitHub confirmation pages, I integrated a custom translation dropdown that toggles between English and Spanish. The dynamic placeholders (student’s name, course list, etc.) are also translated in real-time, giving parents a consistent bilingual experience.

  • Emails: ~25 HTML files for grade-specific and step-specific communications.
  • Static Pages (GitHub): ~15 total, each pulling data via the GET Request Flow.
  • Choice Card Templates: ~20 different HTML/CSS combos used within Power Automate to generate PDFs at each step.

In total, there are over 60 HTML files. Each file typically includes:

Usage & Examples

Core Functionalities

  • Parent/Student Confirmation Pages: Contain dynamic forms to confirm or reject course requests. Data is populated based on query parameters (e.g., ?studentID=12345).
  • Inquiry Pages: Let parents submit phone numbers and comments when requesting changes.

Step-by-Step Walkthrough

  1. Student Submission: The student fills out the Microsoft Form. The Grade Approval Creation Flow triggers.
  2. Student Email: The system emails the student a link to confirm or finalize their choices.
  3. Student Confirmation: Clicking confirm triggers the Student Confirmation Flow, which updates SharePoint, generates a new PDF, and emails the parent.
  4. Parent Email & Pages: The parent sees a bilingual email with a link to either confirm or inquire. That triggers a respective flow (Parent Confirmation Flow or Parent Inquiry Flow).
  5. Final Status: The system logs the final parent response in SharePoint, optionally notifying a counselor for rejections or inquiries.

Example Scenarios

  • Scenario 1: 10th grader picks courses, confirms them, parent also confirms, everything is logged as “Parent Confirmed.”
  • Scenario 2: 8th grader picks courses, confirms, parent requests an inquiry, counselor is notified, modifications are arranged.
  • Scenario 3: A Spanish-speaking parent uses the translation toggle to read all instructions in Spanish and confirm or reject accordingly.

Troubleshooting & FAQ

Common Issues & Solutions

  • Missing API Key for Google Translate: Ensure you’re including the googleTranslateElementInit function and correct script reference. Check the browser’s console for errors.
  • Emails Not Sending: Verify the email API endpoint in your flow or the request body JSON. Test with Postman or a mock URL to confirm payload correctness.
  • Form Data Not Appearing on Pages: Make sure the GET Request Flow is set up with correct query params and that the pages’ JavaScript references the endpoint properly.
  • Browser Compatibility: Some older browsers might not support modern JS features (like fetch); use polyfills or suggest Chrome/Edge.

Debugging Steps

  1. Open the Developer Console (F12) to check for JavaScript errors or missing files.
  2. Verify your Power Automate or Logic App logs to see if any flows are failing silently.
  3. Confirm your SharePoint list has matching fields for the data you expect to store.

FAQs

  • How do I add a new grade? Create a new HTML template for that grade, update your flows’ switch cases, and reference the correct placeholders in the email/page generation steps.
  • Can I change the language options for Google Translate? Yes, update includedLanguages: 'es,en' to the desired languages in your script.
  • How do I customize the CSS or branding? Modify the inline <style> blocks in each template, or link an external .css if you’re hosting them differently.

Challenges & Future Considerations

Complex Branching & Testing

Managing 5 grades, new vs. existing students, bilingual content, and multiple confirmation states (student pending, student confirmed, parent pending, parent confirmed, parent rejected, etc.) required rigorous testing. Each flow had nested conditions, making debugging crucial.

Website vs. Microsoft 365

A centralized custom website might have simplified the architecture but introducing 365-based flows ensured broader adoption within the school. This trade-off is important to consider for future expansions.

Scalability & Maintenance

Over 60 HTML templates can become cumbersome. In the future, consider modularizing repeated parts (like logos, headings, footers) or storing them externally. Also, building an admin dashboard could simplify oversight and reduce confusion among flows.

Enhanced Error Handling & Analytics

Future improvements include more robust error messaging and real-time analytics for how many confirmations are completed daily, potential bottlenecks, and so on.
